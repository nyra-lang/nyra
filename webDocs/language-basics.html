Program structure

A program is a list of instructions the computer runs from top to bottom. In Nyra, the entry point is always a function called main.

fn main() {
    print("Hello, Nyra!")
}
fn main() -> void {
    print("Hello, Nyra!")
}
  • fn main() — defines the function named main. When you run the program, execution starts here.
  • { … } — a block. Code inside the block runs together. Indentation helps you read it; Nyra does not require specific spacing.
  • print(...) — sends text or numbers to the screen (standard output).
  • // comment — notes for humans; the compiler ignores everything after // on that line.

Try it: save as hello.ny, then run nyra run hello.ny. For a folder with main.ny, run nyra run . from that folder.

Variables overview

A variable is a name you give to a value so you can use it later — like labeling a box with what is inside.

let age = 25
print(age)    // prints 25
let age: i32 = 25
print(age)    // prints 25

Here age is the name, and 25 is the value stored under that name. When the program runs, Nyra remembers that age means 25.

Nyra gives you three main ways to introduce names:

  • let — a normal binding; the value should not be replaced later.
  • let mut — a binding you can change later (counters, accumulators, game state).
  • const — a fixed name for a value known when the program is compiled.

let — bind a name (immutable)

let creates a variable and sets its initial value. By default the binding is immutable: after the first assignment, you must not assign a new value to that name.

let player_name = "Nyra"
let score = 10
let pi_approx = 3

print(score)   // OK — reading is always fine
let player_name: string = "Nyra"
let score: i32 = 10
let pi_approx: i32 = 3

print(score)   // OK — reading is always fine

Why immutable by default? When a value never changes, the compiler and other readers of your code can reason about it more easily. Most variables in a program never need to change.

Type inference: Nyra figures out the type from the value. 10 is an i32 (integer). "Nyra" is a string. You can write the type explicitly if you want: let score: i32 = 10.

What if you try to change it? 
let score = 10
score = 20   // compile error — score is not mut
let score: i32 = 10
score = 20   // compile error — score is not mut

Nyra stops at compile time with an error telling you to use let mut if the value must change.

mut — short for mutable

mut is an abbreviation of mutable — meaning the variable is allowed to change after it is created. You use it when the value will be updated later (loops, counters, running totals).

let mut lives = 3
lives = lives - 1    // OK — lives is mutable
print(lives)         // 2
let mut lives: i32 = 3
lives = lives - 1    // OK — lives is mutable
print(lives)         // 2

Think of let mut as: “create a named box, and I will put different values in it over time.”

Shorthand: mut without let

In some examples you will see mut sum = 0 instead of let mut sum = 0. Both declare a mutable variable; the shorter form appears in loops and benchmarks.

fn main() {
    mut counter = 0
    counter = counter + 1
    print(counter)   // 1
}
fn main() -> void {
    mut counter = 0
    counter = counter + 1
    print(counter)   // 1
}

When you need mut

let mut i = 0
while i < 3 {
    print(i)
    i = i + 1        // i must be mut — we change it every loop
}
let mut i: i32 = 0
while i < 3 {
    print(i)
    i = i + 1        // i must be mut — we change it every loop
}

Without mut, the line i = i + 1 would be rejected — you cannot reassign an immutable let.

const — compile-time constants

const defines a constant: a name for a value that is fixed for the entire program and known when you compile (not computed at runtime).

const MAX_PLAYERS = 4
const GRID_SIZE = 8

fn main() {
    print(MAX_PLAYERS)
}
const MAX_PLAYERS = 4
const GRID_SIZE = 8

fn main() -> void {
    print(MAX_PLAYERS)
}

let vs mut vs const — side by side

const MAX = 100          // never changes; known at compile time

let title = "Dungeon"    // set once at runtime; do not reassign
// title = "Other"       // ERROR

let mut gold = 0         // starts at 0; we will add to it
gold = gold + 15         // OK
print(gold)              // 15
const MAX = 100          // never changes; known at compile time

let title = "Dungeon"    // set once at runtime; do not reassign
// title = "Other"       // ERROR

let mut gold: i32 = 0
gold = gold + 15         // OK
print(gold)              // 15

Rule of thumb: use let unless you need to reassign — then use let mut. Use const for shared fixed numbers used in many places.

Heap string follows Move rules; structs with only number fields are auto-Copy. Calls often auto-borrow — see Ownership UX and Memory & ownership.

Types — what kind of value is it?

A type tells Nyra what operations are allowed. You cannot add a string to an integer; the type checker catches mistakes before the program runs.

let n = 0       // integer (32-bit)
let ok = true  // true or false
let msg = "hi"
let n: i32 = 0
let ok = true  // true or false
let msg: string = "hi"

Usually Nyra infers the type so you can omit : i32. Float literals like 3.14 infer f64. Add annotations when it helps readability or the compiler needs a hint.

Functions — reusable pieces of code

A function is a named block of code you can call whenever you need it. Functions avoid repeating the same logic and make programs easier to read.

fn add(a, b) {
    return a + b
}

fn greet(name) {
    print(name)
}

fn main() {
    let total = add(10, 32)
    print(total)      // 42
    greet("Welcome")
}
fn add(a: i32, b: i32) -> i32 {
    return a + b
}

fn greet(name: string) {
    print(name)
}

fn main() -> void {
    let total = add(10, 32)
    print(total)      // 42
    greet("Welcome")
}
  • fn — starts a function definition.
  • add — the function name.
  • (a: i32, b: i32) — parameters: names and types the caller must provide.
  • -> i32 — return type (what the function gives back). Omit or use -> void when there is no return value.
  • return — sends a value back to the caller.

Generics (functions that work on many types) are covered in Generics.

Loops — repeat work

Loops run the same block multiple times. You almost always need let mut for a counter variable.

while — repeat while a condition is true

let mut i = 0
while i < 5 {
    print(i)
    i = i + 1
}
let mut i: i32 = 0
while i < 5 {
    print(i)
    i = i + 1
}

Each time through the loop, i increases by 1 until i is no longer less than 5.

for — iterate a numeric range

for x in start..end runs with x equal to each integer from start up to, but not including, end.

for i in 0..5 {
    print(i)    // prints 0, then 1, 2, 3, 4
}
for i in 0..5 {
    print(i)    // prints 0, then 1, 2, 3, 4
}

Example: sum numbers with while

fn main() {
    mut sum = 0
    mut i = 0
    let n = 10
    while i < n {
        sum = sum + i
        i = i + 1
    }
    print(sum)   // 0+1+2+...+9 = 45
}
fn main() -> void {
    mut sum = 0
    mut i = 0
    let n: i32 = 10
    while i < n {
        sum = sum + i
        i = i + 1
    }
    print(sum)   // 0+1+2+...+9 = 45
}

Run: nyra run examples/comparison/loop/sum_loop.ny (uses a larger n).

Conditionals — choose what runs

if runs one block when a condition is true, optionally another when it is false.

let score = 75

if score >= 60 {
    print("pass")
} else {
    print("fail")
}

let grade = if score >= 90 { "A" } else { "B" }
let score: i32 = 75

if score >= 60 {
    print("pass")
} else {
    print("fail")
}

let grade = if score >= 90 { "A" } else { "B" }

The last line is an if expression: it produces a value ("A" or "B") that you bind with let.

Structs — group related data

A struct groups named fields. In zero-types style, use an object literal directly — no struct keyword. Add a named struct when you need methods (impl), imports, or FFI.

fn main() {
    let mut p = { hp: 100, score: 0 }
    p = { hp: p.hp, score: p.score + 10 }
    print(p.score)   // 10
}
fn main() -> void {
    let mut p = { hp: 100, score: 0 }
    p = { hp: p.hp, score: p.score + 10 }
    print(p.score)   // 10
}

Access fields with a dot: p.score, p.hp. Separate literal fields with commas. See Nyra Structs for object literals vs named struct.

Enums & match — fixed choices

An enum lists allowed variants. match picks code based on which variant you have.

enum Status {
    Idle
    Running
    Done
}

fn code(s) {
    return match s {
        Status.Idle => 0
        Status.Running => 1
        Status.Done => 2
    }
}
enum Status {
    Idle
    Running
    Done
}

fn code(s: Status) -> i32 {
    return match s {
        Status.Idle => 0
        Status.Running => 1
        Status.Done => 2
    }
}

See Match for guards and more patterns.

Imports — split code across files

Large programs use multiple .ny files. import brings names from another file into the current one.

import "src/config.ny"
import "src/engine.ny"

fn main() {
    print(APP_TITLE)
    print(run_sim())
}
import "src/config.ny"
import "src/engine.ny"

fn main() -> void {
    print(APP_TITLE)
    print(run_sim())
}

Paths are relative to the file that contains the import. Run multi-file projects with nyra run . from the folder that contains main.ny.

Methods — functions on a type

struct Counter {
    value: i32
}

impl Counter {
    fn bump(self) {
        return Counter { value: self.value + 1 }
    }
}

fn main() {
    let c = Counter { value: 0 }
    let c2 = c.bump()
    print(c2.value)   // 1
}
struct Counter {
    value: i32
}

impl Counter {
    fn bump(self) -> Counter {
        return Counter { value: self.value + 1 }
    }
}

fn main() -> void {
    let c: Counter = Counter { value: 0 }
    let c2 = c.bump()
    print(c2.value)   // 1
}

impl Counter { ... } attaches methods to Counter. Call them with dot syntax: c.bump().
